# Factory method

Template Patternにおいてオブジェクトを生成する処理だけ決まったパターンが存在する場合、そこだけを抽象化して具体的なオブジェクトの生成方法などを具体的な実装に任せたい場合に採用するデザインパターン

Rustではtraitは具体的な実装を持つことができない事とimplした場合、変えてほしくない関数も変えてしまうことができることから、実行だけしてほしい処理をSealedFactory内で記述して
ユーザーが定義した処理を実行するようにして,オブジェクト生成処理を共通化している

Template Methodでも利用しているが、internalの中でSealedというtrait制約を指定することで提供するImplSealedFactoryは内部的にしか実装することができない制限をかけている
